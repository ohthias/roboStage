O RoboStage continua evoluindo graças ao feedback da comunidade.

Se você encontrou algum problema, teve uma ideia para uma nova ferramenta ou acredita que alguma funcionalidade pode ser melhorada, agora existe um lugar central para registrar tudo isso: o **GitHub Issues**.

## O que pode ser enviado?

O espaço de *Issues* foi organizado para receber diferentes tipos de contribuições, como:

* relatos de bugs;
* sugestões de melhorias;
* propostas de novas ferramentas;
* dúvidas sobre funcionalidades da plataforma;
* solicitações de novos recursos;
* problemas de usabilidade.

Cada contribuição ajuda a identificar prioridades e torna o desenvolvimento do RoboStage mais transparente para toda a comunidade.

Você pode abrir uma nova issue em:

**GitHub Issues**
[Clique aqui!](https://github.com/ohthias/roboStage/issues)

## Encontrou uma falha de segurança?

Se você identificar uma vulnerabilidade de segurança, um problema que possa comprometer dados ou qualquer outra falha relacionada à segurança da plataforma, **evite criar uma Issue pública**.

Nesses casos, utilize o canal de **Security Advisories** do repositório, que permite o envio privado da vulnerabilidade para análise e correção antes da divulgação pública. Esse fluxo segue as boas práticas recomendadas pelo GitHub para projetos de código aberto.

O canal de reporte está disponível em:

**Security**
[Clique Aqui!](https://github.com/ohthias/roboStage/security/policy)

## Sua participação faz diferença

O RoboStage é um projeto desenvolvido de forma independente e está em constante evolução.

Cada bug reportado, sugestão enviada ou melhoria proposta contribui diretamente para tornar a plataforma mais estável, segura e útil para equipes de robótica.

Se você utiliza o RoboStage, sua participação é uma das melhores formas de ajudar no crescimento do projeto. 🚀
